项目名称
控制台交互式游戏原型（纯 C 语言）
项目定位（重点）
这是一个完全基于控制台输出、无任何图形素材、无引擎、纯代码实现的交互式游戏原型。
项目目标不是画面，而是：
通过代码本身，构建一个可交互、可扩展、接近完整游戏逻辑的“箱庭式”系统
核心特点
✅ 纯控制台输出（printf）
✅ 无素材 / 无 UI / 无引擎
✅ 完全由逻辑驱动
✅ 可交互（用户输入影响世界状态）
✅ 箱庭式设计思路（状态随行为变化）
这不是演示程序，而是：
用最低表现形式，验证游戏系统是否成立
当前实现内容
1️⃣ 体力系统
角色拥有体力值（0–100）
不同行为对体力产生不同影响
体力状态决定游戏是否继续
2️⃣ 行为系统
输入
行为
效果
1
休息
恢复体力（有上限）
2
行走
缓慢消耗体力
3
奔跑
快速消耗体力
4
退出
主动结束游戏
3️⃣ 游戏循环
使用 while 构建持续运行的游戏主循环
每一轮循环代表一次“决策”
玩家行为 → 状态变化 → 世界反馈
这是典型游戏主循环的最小实现形式
设计思想说明（重点）
本项目刻意：
❌ 不使用图形
❌ 不追求表现
❌ 不依赖任何第三方库
目的只有一个：
把“游戏”拆解成：状态 + 行为 + 规则 + 反馈
这套逻辑：
可以无缝迁移到 SDL / Unity / Godot
可以扩展成 RPG / 生存 / 策略 / 模拟类游戏
本质上就是“游戏引擎最底层的那一层”
学习与能力体现
通过该项目，已经实际练习并掌握：
while 构建持续系统
switch-case 处理行为分发
状态边界控制（体力不溢出、不为负）
游戏逻辑的因果关系设计
